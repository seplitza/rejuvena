# Инструкция по управлению тегами RU

## Выполненные изменения

### 1. Модель Tag
- Добавлено поле `isVisible` (boolean, default: true) для управления видимостью тегов
- Теги с `isVisible: false` не отображаются в API `/tags`, но продолжают работать в фильтрации

### 2. Создан тег RU
- Тег `RU` автоматически создан и помечен как скрытый (`isVisible: false`)
- Добавлен всем 63 существующим упражнениям
- Этот тег предназначен для фильтрации упражнений по языку во фронтенде

### 3. Обновлены скрипты импорта
Следующие скрипты теперь автоматически добавляют тег `RU` при импорте:
- `import-forehead-eyes-pro.ts`
- `import-cheeks-eyes-manual.ts`
- `import-cheeks-eyes-pro.ts`
- `import-neck-pro-manual.ts`

### 4. Создан вспомогательный модуль
- `src/scripts/utils/ru-tag.ts` - функция `getRuTag()` для получения/создания тега RU
- Используется во всех скриптах импорта для единообразия

### 5. Обновлен API endpoint
- GET `/tags` теперь возвращает только видимые теги (isVisible !== false)
- Скрытые теги (как RU) не попадают в ответ API

## Как использовать

### При импорте новых упражнений на русском языке:
Все скрипты импорта уже обновлены и автоматически добавляют тег `RU`. Ничего дополнительного делать не нужно.

### Для ручного добавления тега RU к упражнениям:
```bash
npx ts-node src/scripts/add-ru-tag-all.ts
```

### Во фронтенде (фильтрация по языку):
Теперь можно фильтровать упражнения по тегу `RU`:
```typescript
// Получить только русские упражнения
const russianExercises = exercises.filter(ex => 
  ex.tags.some(tag => tag.name === 'RU')
);
```

## Примечания
- Тег `RU` не отображается в списке тегов API
- Тег `RU` не показывается в админ-панели при редактировании (фильтруется на бэкенде)
- Все последующие импорты автоматически получают тег `RU`
- Теги "Продвинутый" и "продвинутое" уже были удалены ранее (дублируют PRO)
